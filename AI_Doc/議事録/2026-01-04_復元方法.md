# 現状の復元手順 (2026-01-04)

## 1. 開発環境の復元
プロジェクトをクローンまたはバックアップから復元した際、以下の手順で実行環境を構築します。

1. **Gitリポジトリの確認**
   - 2026-01-04 時点で `git init` を行い、現在のスナップショットを保存済みです。
2. **依存パッケージのインストール**
   ```bash
   npm install
   ```
3. **開発サーバーの起動**
   ```bash
   npm run dev
   ```

## 2. データの健全性確認
画面が真っ暗になる、あるいは進行不能になる場合は、以下のJSONデータの紐付けを再確認してください。

- **`src/assets/data/episodes.json`**
  - 各エピソードの `startStoryID` が `scenario.json` に存在するか確認。
- **`src/assets/data/scenario.json`**
  - `storyID` の重複がないか、また選択肢 (`CHOICE`) や遷移 (`TAP_NEXT`) の `nextStoryID` が正しく設定されているか確認。
- **`src/assets/data/gallery.json`**
  - 画像パスや各画像の `id` がフロントエンドの参照と一致しているか確認。

## 3. 実装上の注意 (現在)
- **Asset Import 画面**: 現在はモックアップの状態であり、実際のDropbox連携にはAPI Keyの設定とロジックの実装が必要です。
- **永続化**: 現在の状態管理（Zustand）はメモリ上のみであり、リロードで初期化されます。セーブ/ロード機能の実装が今後の課題です。
